# r2 ulozisko

## lokalny vyvoj (predvolene)

pouzite predvolene lokalne disky:

```dotenv
FILES_DISK=public
FILES_PRIVATE_DISK=local
```

spustite healthcheck z adresara backend:

```bash
cd backend
php artisan storage:r2-healthcheck
```

toto by malo lokalne prejst.

## zapnutie r2 lokalne (volitelne)

nastavte tieto env premenne, ak chcete testovat cloudflare r2 na lokalnom pocitaci:

```dotenv
FILES_DISK=r2_public
FILES_PRIVATE_DISK=r2_private
AWS_ACCESS_KEY_ID=__R2_ACCESS_KEY_ID__
AWS_SECRET_ACCESS_KEY=__R2_SECRET_ACCESS_KEY__
AWS_DEFAULT_REGION=auto
AWS_BUCKET=__R2_PUBLIC_BUCKET__
AWS_URL=
AWS_ENDPOINT=https://__ACCOUNT_ID__.r2.cloudflarestorage.com
AWS_USE_PATH_STYLE_ENDPOINT=true
R2_PRIVATE_BUCKET=__R2_PRIVATE_BUCKET__
```

poznamka k `AWS_URL`:

- ak vlastna domena este neexistuje, nechajte `AWS_URL` prazdne
- ak bude neskor dostupna vlastna domena, napr. `https://media.astrokomunita.sk`, nastavte ju do `AWS_URL`, aby `Storage::url()` vracalo krajsie url adresy

## nastavenie pre produkciu/staging

pouzite tuto env sablonu:

```dotenv
FILES_DISK=r2_public
FILES_PRIVATE_DISK=r2_private
AWS_ACCESS_KEY_ID=__R2_ACCESS_KEY_ID__
AWS_SECRET_ACCESS_KEY=__R2_SECRET_ACCESS_KEY__
AWS_DEFAULT_REGION=auto
AWS_BUCKET=__R2_PUBLIC_BUCKET__
AWS_URL=
AWS_ENDPOINT=https://__ACCOUNT_ID__.r2.cloudflarestorage.com
AWS_USE_PATH_STYLE_ENDPOINT=true
R2_PRIVATE_BUCKET=__R2_PRIVATE_BUCKET__
```

varovanie:

- nikdy necommitujte `backend/.env`
- nikdy necommitujte access keys ani ine realne prihlasovacie udaje

potom spustite:

```bash
cd backend
php artisan optimize:clear
php artisan storage:r2-healthcheck
```

`php artisan optimize:clear` je preferovana volba, pretoze naraz vycisti cache pre config, routes aj views. ak treba, ako alternativa stale funguje aj kombinacia `php artisan config:clear` a `php artisan cache:clear`.

overte v cloudflare ui:

- verejny bucket: `healthchecks/public/...`
- sukromny bucket: `healthchecks/private/...`

## server deploy checklist

- nastavte env premenne
- spustite `php artisan optimize:clear`
- spustite `php artisan storage:r2-healthcheck`
- overte v cloudflare ui, ze sa objekty objavia pod `healthchecks/public/...`
- overte v cloudflare ui, ze sa objekty objavia pod `healthchecks/private/...`
